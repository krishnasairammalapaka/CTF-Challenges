Forensics Challenge

Objective

You are given a disk image forensic_image.dd, which contains evidence of deleted files. Your task is to use digital forensics tools to analyze the disk image, recover any deleted files, and extract a hidden flag from them.

Challenge Description

You have been provided with a raw disk image file, forensic_image.dd, which was obtained from a compromised system. The disk image is believed to contain deleted files, and potentially hidden data that could point to an important flag for the CTF competition.

Your goals:

  1. Analyze the disk image for any traces of deleted files.
  2. Recover the deleted files that might contain hidden or important information.
  3. Extract the hidden flag from the recovered files.

Recommended Tools

  • Autopsy: A digital forensics platform for analyzing disk images
  • SleuthKit: Command-line tools for analyzing and recovering data
  • Foremost: A tool for file carving
  • ddrescue: (optional) For recovering corrupted sectors

Detailed Instructions

1. Mount the disk image

sudo mount -o loop forensic_image.dd /mnt/forensic_image

2. Examine the file system

ls /mnt/forensic_image

3. Identify deleted files

fls -r forensic_image.dd

4. Recover deleted files

foremost -i forensic_image.dd -o recovered_files/

5. Search for the flag

grep -r "flag{" recovered_files/

6. Check for hidden data

binwalk recovered_image.png

Hints for Success

  • Pay special attention to files marked as deleted in Autopsy/SleuthKit
  • Remember that files may be fragmented across the disk
  • Look for common file signatures (JPEG: FFD8, PNG: 89504E47)
  • Check recovered files for steganography
  • Use grep to scan for flag patterns

Expected Flag Format

The flag will be in the following format:

flag{this_is_the_flag_you_found_in_the_recovered_file}
