Challenge 1: Corporate Infrastructure Breach

Scenario:

You're tasked with penetrating a corporate network that implements defense-in-depth. The network includes Active Directory, network segmentation, IDS/IPS systems, and various security controls. Your goal is to achieve domain admin access and locate the flag in a secured database.

Environment Access:

Initial access is provided through a VPN connection to a simulated corporate network:

VPN Endpoint: vpn.challenge.local
Initial Access: Compromised employee workstation (10.0.5.15)
Target Domain: CORP.CHALLENGE.LOCAL

Your Tasks:

  1. Perform initial enumeration without triggering IDS alerts
  2. Exploit Active Directory misconfigurations
  3. Achieve lateral movement while evading detection
  4. Escalate privileges to Domain Admin
  5. Access the secured database and retrieve the flag

Provided Resources:

  • Network Diagram
  • VPN Configuration
  • Initial Access Credentials

Challenge 2: Smart Building System Compromise

Scenario:

A smart building management system controls critical infrastructure including HVAC, access control, and surveillance. Your task is to compromise the system by exploiting various IoT vulnerabilities while avoiding detection by the building's security team.

Target Environment:

Management Interface: https://bms.challenge.local
IoT Protocol Stack: MQTT, CoAP, Modbus TCP
Security Controls: TLS, Device Authentication, Firmware Signing

Your Tasks:

  1. Analyze and exploit firmware vulnerabilities
  2. Bypass device authentication mechanisms
  3. Exploit protocol-specific vulnerabilities
  4. Gain control of the central management system
  5. Extract the flag from the secure storage

Provided Files:

  • IoT Device Firmware
  • Protocol Specifications
  • Device Documentation

Challenge 3: Cloud Platform Infiltration

Scenario:

Target is a multi-cloud environment using AWS and Azure services. The infrastructure implements various security controls including IAM, network security groups, and WAF. Your goal is to compromise the environment and locate the flag in a secured S3 bucket.

Initial Access:

Compromised Credentials: AWS IAM User (Limited Permissions)
Web Application: https://app.challenge.cloud
CI/CD Pipeline: Jenkins instance accessible internally

Your Tasks:

  1. Enumerate cloud resources without triggering alerts
  2. Exploit misconfigurations in cloud services
  3. Perform privilege escalation in the cloud environment
  4. Bypass WAF and security controls
  5. Access the secured S3 bucket containing the flag

Provided Materials:

  • Cloud Architecture Diagram
  • Initial IAM Policy
  • AWS Credentials

Challenge 4: Software Supply Chain Compromise

Scenario:

A software company uses a complex CI/CD pipeline for their application deployment. Your goal is to compromise the pipeline, inject malicious code, and extract the flag from a production system without being detected.

Environment Details:

Source Code: Private GitHub repository
CI/CD: GitLab CI with custom security checks
Deployment: Kubernetes cluster with Pod Security Policies
Security: Code signing, container scanning, runtime security

Your Tasks:

  1. Analyze the CI/CD pipeline for vulnerabilities
  2. Bypass code signing and security checks
  3. Inject malicious code into the build process
  4. Evade runtime security controls
  5. Extract the flag from production environment

Provided Resources:

  • CI/CD Configuration
  • Kubernetes Security Policies
  • Build Scripts
